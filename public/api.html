Convert To It Free API (v1)

This API is for reading format metadata (what can be converted), not for uploading files. If you want to convert a file right now, use the main converter UI.

Not technical? Use the converter page instead of the API.

  • No API key
  • Read-only metadata
  • GET / HEAD / OPTIONS
  • Rate limit: 60 requests / 60s / IP

API in plain English

What you can do

  • Check API status and version
  • List supported input/output formats
  • See which handlers power each format

What you cannot do

  • No direct file conversion API endpoint
  • No authentication or user account endpoints
  • No write/update/delete operations

Who this is for

  • Developers building format pickers
  • Automation scripts that validate format support
  • Teams that want live metadata from converttoit.com

Quick start (3 steps)

  1. Check service status: 
    curl -s https://converttoit.com/api/v1/status | jq
  2. List image formats: 
    curl -s "https://converttoit.com/api/v1/formats?category=image" | jq
  3. See available handlers: 
    curl -s https://converttoit.com/api/v1/handlers | jq

Rate limits (exact behavior)

The API limiter is enforced inside the worker and keyed by cf-connecting-ip. All /api/v1/* endpoints share one bucket.

Quota

60 requests per 60 seconds for each client IP.

The window starts on your first request and resets 60 seconds later.

Which methods count

  • GET: counted
  • HEAD: counted
  • OPTIONS preflight: not counted

When the limit is hit

Server responds with 429 and JSON error rate_limited.

Use the retry-after header to decide when to retry.

Headers returned on 429 responses

retry-after
Seconds to wait before retrying.
x-ratelimit-limit
Current limit value (60).
x-ratelimit-remaining
Always 0 on 429.
x-ratelimit-source
Current source is isolate.
curl -i https://converttoit.com/api/v1/status

Endpoints

These endpoints are read-only and share the same rate-limit bucket described above.

GET /api/v1/status

Returns basic service health, API version, and environment metadata.

Use this when: you want a quick uptime or smoke check.

curl -s https://converttoit.com/api/v1/status | jq

GET /api/v1/formats

Returns supported formats aggregated across handlers.

Use this when: you need dynamic dropdowns or validation logic.

category
Filter by format category (for example: image, audio, video).
from
true = only valid source/input formats.
to
true = only valid target/output formats.
q
Text search by extension, MIME, or format name.
curl -s "https://converttoit.com/api/v1/formats?category=image&from=true&q=png" | jq

GET /api/v1/handlers

Returns handler definitions and the formats each handler supports.

Use this when: you need debugging or capability transparency.

curl -s https://converttoit.com/api/v1/handlers | jq
